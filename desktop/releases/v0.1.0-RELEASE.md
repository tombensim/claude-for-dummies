# Claude for Beginners — Desktop App v0.1.0

**Release Date:** 2026-02-18
**Status:** Internal Alpha
**Platform:** macOS (Electron + Next.js 16)

---

## Overview

First release of the Claude for Beginners desktop application. A guided, beginner-friendly Electron app that wraps Claude Code CLI to help non-technical users build websites through conversation. Supports Hebrew (RTL, default) and English.

---

## Features

### 1. Project Management

- **Automatic project creation** — creates a directory under `~/Documents/Claude Projects/<slug>/` with `git init`
- **Smart slug generation** — strips EN/HE stop words, takes first 3 meaningful words, deduplicates with `-2`, `-3` suffix
- **Persistent metadata** — project state (name, locale, vibe, sessionId, liveUrl) stored via `electron-store`
- **Project switcher** — welcome screen shows recent projects sorted by last opened, click to resume
- **Session resumption** — reopening a project restores the Claude session ID for continuity

### 2. Chat Interface

- **Activity block grouping** — consecutive tool calls merged into collapsible "building" blocks showing file count
- **Markdown rendering** — assistant messages rendered with `react-markdown` + `remark-gfm` (code blocks, links, lists)
- **Live activity bar** — animated status indicator during streaming (e.g., "Creating index.tsx...", "Installing packages...")
- **Chat persistence** — messages saved to `.cc4d/chat-history.json`, restored on app reopen
- **Conversation recovery banner** — auto-dismissing notification when a previous session is restored
- **Component architecture** — ChatPanel decomposed into UserMessage, AssistantMessage, BuildingBlock, ErrorMessage, ChatInput, ChatHistory, LiveActivityBar, ConversationRecoveryBanner

### 3. Agent Communication

- **SSE streaming** — real-time event parsing from Claude CLI via `/api/agent` route
- **Tool-aware status messages** — Bash, Write, Edit, git, shadcn commands produce friendly progress text
- **Session ID capture** — `onSessionId` callback persists session across restarts
- **Raw event forwarding** — `onRawEvent` callback feeds debug terminal
- **Locale-aware messages** — status text adapts to Hebrew/English

### 4. Guided Build Flow (cc4d Skill)

- **Skill bundling** — `skills/cc4d/` copied into project `.cc4d/` on creation
- **Progress tracking** — `.cc4d-progress.json` tracks completed steps
- **CLAUDE.md generation** — skill instructions written to project root, paths rewritten for local execution
- **Step indicator** — visual progress bar in build page header

### 5. Logging Infrastructure

- **4-file logging system** — electron.log, agent.log, app.log, renderer.log under `~/.claude-for-beginners/logs/`
- **Renderer log forwarding** — `console.error` and `console.warn` patched to forward to main process via IPC
- **Log rotation** — 5 MB max, 5 files rotation; electron.log uses `.old` backup
- **Dev convenience** — `npm run logs` tails all logs, `npm run debug:status` shows file sizes

### 6. Debug Tools

- **Debug terminal** — `Cmd+Shift+D` toggles overlay showing raw Claude CLI events
- **Color-coded events** — each event type (system, assistant, tool_use, result, error) has distinct color
- **Expandable rows** — click to see full JSON payload
- **Filtering** — text filter across all event fields
- **Auto-scroll** — follows latest events with manual override

### 7. Welcome Experience

- **Single-screen start** — simplified from multi-step onboarding to one-click "Start building!" button
- **Recent projects** — horizontal scrollable cards with relative timestamps ("just now", "yesterday", "3 days ago")
- **Bilingual UI** — full Hebrew RTL and English support with `next-intl`

### 8. Electron Shell

- **Window management** — 1200x800 default, 900x600 minimum, draggable title bar region
- **External links** — `shell:open-external` IPC handler for safe link opening
- **Bundled dependencies** — Node.js, Git (dugite), and cc4d skill packaged as extraResources
- **Auto DevTools** — developer tools open automatically in non-packaged builds

## Architecture

```
desktop/
├── electron/
│   ├── main.js              # Electron main process (IPC, window, project mgmt)
│   ├── preload.js           # Context bridge (window.electronAPI)
│   ├── preload.ts           # Type declarations for electronAPI
│   ├── project-store.js     # electron-store wrapper for project metadata
│   └── project-utils.js     # slugifyIdea, deduplicatePath helpers
├── src/
│   ├── app/
│   │   ├── api/agent/route.ts   # Next.js API route → Claude CLI spawn
│   │   ├── welcome/page.tsx     # Project selection / creation
│   │   ├── build/page.tsx       # Main chat + build interface
│   │   └── ship/page.tsx        # Deployment flow
│   ├── components/
│   │   ├── chat/                # 9 chat sub-components
│   │   ├── debug/               # Debug terminal overlay
│   │   ├── brand/               # Logo, LanguageToggle
│   │   └── progress/            # StepIndicator, PhaseLabel
│   └── lib/
│       ├── store.ts             # Zustand state (messages, project, streaming)
│       ├── agent-client.ts      # SSE client + event parser
│       ├── activity-blocks.ts   # Message → activity block grouping
│       ├── use-chat-persistence.ts  # Load/save hook
│       ├── renderer-logger.ts   # Console patching for log forwarding
│       ├── debug-store.ts       # Debug terminal state
│       └── use-debug-shortcut.ts    # Cmd+Shift+D handler
├── messages/
│   ├── en.json                  # English translations
│   └── he.json                  # Hebrew translations
└── __tests__/                   # Unit, component, and E2E tests
```

## Known Limitations (v0.1.0)

1. **macOS only** — no Windows/Linux testing or packaging yet
2. **No live preview** — browser preview panel removed in favor of single-column chat
3. **No deployment integration** — ship page exists but deployment flow not wired to a provider
4. **No auto-update** — no electron-updater configured
5. **No code signing** — macOS builds will trigger Gatekeeper warnings
6. **Claude CLI required** — expects `~/.local/bin/claude` to be installed separately
7. **Port conflict** — Next.js dev server on 3456 may conflict if not cleaned up on exit
8. **No offline mode** — requires internet for Claude CLI communication
